# Inspiration — scope note

The files in this folder are **loose visual inspiration**: moodboard references for the
*look and feel* of the NC-750 / Lab visual language (machined enclosures, exploded
engineering diagrams, spec-sheet annotation, one loud accent on neutral linework). They are
**not** philosophical sources and carry no doctrinal weight. The binding documents are
`../BRAND.md`, `../VISUAL_IDENTITY.md`, and `../ETHOS.md`.

## Specifically: `machine_documentation/01.jpg` ("Making Software")

This image is kept for its **visual** contribution only — software drawn as an exploded,
leader-line engineering diagram. Its *textual thesis* ("knowing how things work… today we
understand them in a shallow, abstracted way… there's nothing actionable here, you just need
to be curious") is **not** a brand value and must not be treated as one. The brand's stance is
**utility, not curiosity-for-its-own-sake**: a feature earns its place by helping the user get
work done, and a product's internals may be hidden when hiding them serves the task (see
`../VISUAL_IDENTITY.md` P5). Do not reintroduce a "deep-tool-understanding vs. shallow
abstracted use" framing into any brand document on the strength of this image.

This note exists because that thesis was, at one point, read off this image and into the
brand analysis as if it were intent; it is not. Visual inspiration only.
